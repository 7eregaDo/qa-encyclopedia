## The Intent
The interviewer is checking whether you understand that running a performance test without knowing what to measure is pointless. They want to see that you can connect each metric to a specific quality question and that you know what a bad number looks like.

## The STAR Script
**Situation:** After my team ran a load test ahead of a product launch, I was asked to write up the results for the project manager, who was not technical.

**Task:** I needed to translate the raw JMeter output — which included dozens of columns — into a clear summary that explained what each key metric meant and whether we had passed.

**Action:** I identified the five metrics that mattered for our SLA: average response time (target: under 2 s), 90th percentile response time (target: under 3 s), error rate (target: under 1%), throughput (target: at least 100 requests/second), and CPU utilisation on the server (target: under 80%). I wrote a one-paragraph explanation of each and marked each one pass or fail against the agreed threshold.

**Result:** The project manager was able to sign off confidently and the report was used as a template for future releases.

## Key Terms
| Term | Definition |
|------|------------|
| **Average response time** | The mean elapsed time for all requests in a test run — useful as a summary but sensitive to outliers |
| **Percentile response time** | The response time at or below which a given percentage of requests fall (e.g., the 95th percentile is the threshold below which 95% of requests complete) |
| **Error rate** | The percentage of requests that return an error response; elevated error rates indicate the system is failing under load |
| **Throughput** | The number of requests processed per second; a measure of system capacity |
| **CPU utilisation** | The percentage of processor capacity consumed by the server during the test; sustained high CPU indicates a capacity or efficiency problem |
| **Memory utilisation** | The amount of RAM consumed over time; a steady upward trend during a soak test suggests a memory leak |
| **Apdex score** | Application Performance Index — a standardised 0–1 score that categorises requests as satisfactory, tolerable, or frustrating based on configurable time thresholds |
