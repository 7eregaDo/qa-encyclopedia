## The Intent
The interviewer is not expecting you to be an expert in every tool. They want to see that you are aware of the ecosystem, can name tools with some rationale, and can describe at least one you have used — demonstrating that you know performance testing is a practical discipline, not a theoretical one.

## The STAR Script
**Situation:** I was self-studying for a QA role that listed performance testing experience as desirable. I had never run a formal performance test before.

**Task:** I needed to choose a tool to learn, understand why it was appropriate, and produce a working example to include in my portfolio.

**Action:** I compared JMeter (GUI-based, mature, protocol-rich), Gatling (code-based, Scala, strong reporting), k6 (JavaScript-based, CI-friendly, modern developer workflow), and Locust (Python-based, scriptable, easy ramp-up). Given my existing JavaScript knowledge, I chose k6 and wrote a load test against a public demo API. I scripted a 50-VU ramp-up over 30 seconds, a 2-minute steady state, and a ramp-down, then captured the output summary.

**Result:** I had a working k6 script and result summary in my portfolio, and I could speak to why I chose k6 over alternatives — which came up directly in my next interview.

## Seniority Shift
| Level | Expected Depth |
|-------|---------------|
| **Entry** | Can name two or three tools, explain their key characteristics, and describe which one they have used |
| **Mid** | Selects tools based on team tech stack, CI/CD integration requirements, and protocol needs (HTTP, WebSocket, gRPC) |
| **Senior** | Owns the tooling decision across the organisation, evaluates cloud-based platforms (e.g., BlazeMeter, Grafana Cloud k6), and integrates performance gates into pipelines |

## Key Terms
| Term | Definition |
|------|------------|
| **Apache JMeter** | An open-source Java-based performance testing tool with a GUI, wide protocol support, and a large plugin ecosystem |
| **Gatling** | A code-first performance tool with a Scala/Java DSL, known for high-throughput simulation and detailed HTML reports |
| **k6** | A modern developer-oriented load testing tool scripted in JavaScript, designed for CI/CD integration and cloud execution |
| **Locust** | A Python-based load testing tool where user behaviour is defined in plain Python, suitable for teams with Python expertise |
| **BlazeMeter** | A cloud-based performance testing platform that can run JMeter and Gatling scripts at scale |
| **Protocol support** | The network protocols a tool can test; most tools cover HTTP/HTTPS, while some additionally support WebSocket, gRPC, JDBC, or MQTT |
| **Distributed testing** | Running a load test across multiple machines to generate higher VU counts than a single machine can sustain |
