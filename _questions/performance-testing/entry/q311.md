## The Intent
The interviewer is checking whether you understand that quality extends beyond correctness — a system can pass every functional test and still fail users by being too slow, too fragile under load, or too expensive to run. They want to see that you treat performance as a testable, measurable property, not a vague feeling.

## The STAR Script
**Situation:** During my first placement on a QA team at an e-commerce company, I noticed that the regression suite covered all functional flows but nothing related to speed or load.

**Task:** I was asked to investigate what performance testing meant in practice and where it would fit into our process.

**Action:** I researched the main performance test types — load, stress, spike, and soak — and documented definitions alongside the metrics each one targets (response time, error rate, throughput). I then ran a simple load test using JMeter against our staging login endpoint with 50 concurrent users to demonstrate what baseline data looked like.

**Result:** The team adopted a basic performance baseline check as part of the release checklist. I came away with a clear understanding that performance testing answers the question: "Does this system behave acceptably when real users use it?"

## Seniority Shift

## Key Terms
| Term | Definition |
|------|------------|
| **Response time** | The elapsed time between a user action and the system's response |
| **Throughput** | The number of requests a system processes per unit of time, typically requests per second |
| **Latency** | The delay before data transfer begins; a component of response time |
| **Concurrency** | The number of users or threads interacting with the system simultaneously |
| **SLA (Service Level Agreement)** | A contractual or internal commitment defining acceptable performance thresholds |
| **Baseline** | A recorded set of performance metrics under normal conditions, used as a reference for comparison |
| **Non-functional testing** | Testing that validates system attributes beyond correctness — including performance, security, and usability |
