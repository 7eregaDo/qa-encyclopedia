## The Intent
The interviewer is checking whether you understand that a performance number has no meaning without context. "The page loads in 1.8 seconds" tells you nothing unless you know whether that is better or worse than before. A baseline transforms a measurement into a comparison — which is the foundation of meaningful performance analysis.

## The STAR Script
**Situation:** My team noticed that the checkout page had become noticeably slower after three sprints of backend work, but no one could quantify the degradation because no historical data existed.

**Task:** I was asked to establish a performance testing practice so that future regressions could be detected and quantified.

**Action:** I ran a load test against the current state of staging — 100 VUs, 10-minute steady state — and recorded the key metrics: average response time (2.1 s), 95th percentile (3.4 s), error rate (0.3%), and throughput (48 req/s). I documented the test conditions (environment spec, data set, VU count) alongside the results and stored them in the project wiki as the official baseline.

**Result:** At the next sprint review, a re-run of the same test showed the 95th percentile had climbed to 4.9 s. With the baseline in place, the team could demonstrate a 44% degradation and prioritise a performance fix. Without it, the conversation would have remained anecdotal.

## Key Terms
| Term | Definition |
|------|------------|
| **Baseline** | A recorded set of performance metrics under defined conditions, used as the reference for future comparisons |
| **Performance regression** | A measurable degradation in system performance compared to a previously recorded baseline |
| **Test conditions** | The documented parameters of a test run — including environment spec, data set, VU count, and ramp-up profile — required to make a baseline reproducible |
| **Benchmark** | A target performance level, often derived from an SLA or competitive analysis, against which the system is measured |
| **Trend analysis** | Tracking performance metrics across multiple releases over time to identify gradual degradation before it becomes critical |
| **Environment parity** | The requirement that baseline and comparison tests run in environments with equivalent hardware, configuration, and data volume |
| **Delta** | The measured difference between a current result and the baseline, typically expressed as an absolute value or percentage |
