It is preferable to automated testing when the test involves human judgment, visual inspection, exploratory investigation, or when the cost of automation exceeds its benefit — such as for one-off tests, rapidly changing UI, or usability evaluation."

## The Intent

The interviewer is checking whether you understand that manual and automated testing are complementary approaches — not competing ones. They want to see that you can articulate when each is the right tool, and that you don't treat automation as universally superior. A candidate who says "manual testing should eventually be replaced by automation" is missing significant nuance.

---

## The STAR Script

**Situation:** On a project with a rapidly evolving UI — a new design system was being rolled out incrementally — we had a mix of automated E2E tests and manual testing responsibilities.

**Task:** I needed to decide which test activities were best suited to manual execution versus automation, given the pace of UI change and the nature of the features being tested.

**Action:** I evaluated each test activity against three questions: Does it require human judgment to assess correctness? Is it stable enough to automate without high maintenance cost? Is it executed frequently enough that automation would save meaningful time? For visual layout and usability checks — does the page feel intuitive, is the spacing consistent, does the copy make sense in context — manual testing was clearly preferable. These involved judgment calls that automated assertions could not replicate. For the new design system rollout, UI components were changing weekly, meaning automated selectors would break constantly. Manual exploratory testing was far more efficient during this phase. For stable, repetitive workflows like form submission validation and navigation checks, I flagged these as automation candidates once the UI stabilised.

**Result:** By directing manual effort toward judgment-dependent and high-churn areas, and reserving automation for stable, repetitive scenarios, the team reduced wasted automation maintenance time by approximately 40% and improved exploratory coverage of the design system rollout significantly. Three usability issues were found through manual exploration that no automated test would have detected.

## Key Terms

| Term | Definition |
|------|------------|
| **Manual Testing** | Test execution performed by a human tester without automation tooling. |
| **Automated Testing** | Test execution performed by scripts or tools without direct human intervention during the run. |
| **Exploratory Testing** | A manual technique where test design and execution happen simultaneously, guided by tester judgment. |
| **Usability Testing** | Manual evaluation of how intuitive and user-friendly a system is — requires human perception. |
| **Automation Candidate** | A test case that is stable, repetitive, and deterministic — a good fit for automation investment. |
| **Human Judgment** | Assessment that requires perception, intuition, or contextual understanding — the core value of manual testing. |
