category: manual-testing

theory: "Equivalence partitioning is a black-box test design technique that divides a system's input domain into groups — or partitions — where all values within a group are expected to behave identically. By testing one representative value per partition rather than every possible input, testers achieve meaningful coverage with a minimal number of test cases."
---

## The Intent

The interviewer is checking that you can design tests systematically rather than arbitrarily. Equivalence partitioning is one of the most fundamental test design techniques, and knowing it signals that you approach testing with structure and efficiency. They also want to see that you can apply it practically — not just define it.

---

## The STAR Script

**Situation:** I was testing an age validation field for an online service that required users to be between 18 and 65 years old to register. Without a structured approach, I could theoretically test thousands of individual age values.

**Task:** I needed to design a minimal, effective set of test cases that gave meaningful coverage of the validation logic without testing every possible age.

**Action:** I applied equivalence partitioning by identifying the distinct input partitions for the age field. There were three valid and invalid groups. First, **below the valid range**: any age below 18 should be rejected — I chose 10 as a representative value. Second, **within the valid range**: any age from 18 to 65 should be accepted — I chose 35 as a representative value. Third, **above the valid range**: any age above 65 should be rejected — I chose 70 as a representative value. I also identified an additional invalid partition for **non-numeric input** — entering letters or symbols — choosing "abc" as the representative value. This gave me four test cases covering the full input domain, rather than 66 individual numeric tests. Each test case had a clear expected result: accepted or rejected with a specific error message.

**Result:** The four test cases covered all meaningful input scenarios efficiently. The test for below-range input revealed that the form accepted age 17 without error — a validation defect. The non-numeric test revealed that entering letters caused the form to submit without any validation response, producing a server-side error. Both were found with a deliberately minimal test set.

## Key Terms

| Term | Definition |
|------|------------|
| **Equivalence Partitioning** | A test design technique dividing inputs into groups where all values are expected to behave identically. |
| **Partition** | A distinct group of input values expected to produce the same system behaviour. |
| **Valid Partition** | A group of inputs the system should accept and process correctly. |
| **Invalid Partition** | A group of inputs the system should reject with an appropriate error response. |
| **Representative Value** | The single input value chosen from a partition to represent the entire group in a test case. |
| **Input Domain** | The full set of possible inputs for a given field or function. |
