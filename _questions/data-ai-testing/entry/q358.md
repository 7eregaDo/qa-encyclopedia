## The Intent
The interviewer is not expecting expert-level proficiency in every tool. They want to see that you are aware of the ecosystem, can name tools with rationale, and have used at least one — demonstrating that your understanding is grounded in practice, not theory.

## The STAR Script
**Situation:** When I was preparing to move into a QA role on a data-heavy product, I realised my testing toolkit was entirely oriented toward web application testing — Selenium, Postman, JMeter. I needed to understand what tools data and AI teams used and why.

**Task:** I conducted a self-directed learning exercise to map the data and AI testing tooling landscape and identify one tool to learn hands-on.

**Action:** I mapped the tools by layer. For data pipeline quality, I found Great Expectations — an open-source Python library where you define "expectations" (assertions) about your data, such as column completeness or value range constraints, and run them as a test suite against any dataset. For ML model evaluation, scikit-learn provided standard metric functions (accuracy, precision, recall, F1) out of the box, and Evidently AI offered richer model monitoring reports covering drift detection and data quality side by side. For LLM evaluation, I found RAGAS (for retrieval-augmented generation quality) and LangSmith (for tracing and evaluating LLM chains). I chose Great Expectations to learn first — it was the most applicable to the data pipeline work on my target team. I worked through the documentation and wrote a basic expectation suite against a CSV dataset, covering null checks, value ranges, and column presence.

**Result:** I could speak confidently to the tooling landscape in interviews and had a working example in my portfolio. On joining the team, Great Expectations was already partially deployed and I was able to contribute to the expectation suite in my first sprint.

## Key Terms
| Term | Definition |
|------|------------|
| **Great Expectations** | An open-source Python library for defining, documenting, and running data quality assertions (expectations) against datasets and pipelines |
| **dbt tests** | Built-in and custom data quality tests in the dbt data transformation framework — including not-null, unique, accepted-values, and relationship checks |
| **Evidently AI** | An open-source ML monitoring and evaluation library that generates reports on model performance, data drift, and data quality |
| **RAGAS** | An evaluation framework for retrieval-augmented generation (RAG) pipelines, measuring metrics such as faithfulness, answer relevance, and context precision |
| **LangSmith** | A platform from LangChain for tracing, evaluating, and monitoring LLM application behaviour in development and production |
| **scikit-learn metrics** | A module within the scikit-learn Python library providing standard classification and regression evaluation functions |
| **Expectation suite** | In Great Expectations, a named collection of assertions about a dataset — analogous to a test suite in traditional software testing |
