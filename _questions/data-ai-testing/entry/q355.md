## The Intent
The interviewer is checking whether you understand that AI quality is not a one-time validation event. Unlike traditional software, which only changes when code changes, an ML model can degrade without any code being touched — because the world it was trained to understand keeps changing. A QA engineer who understands drift treats monitoring as part of the quality responsibility.

## The STAR Script
**Situation:** Our product included a fraud detection model that had been trained 14 months ago and had passed extensive pre-launch testing. A data analyst noticed that the fraud flag rate had dropped from 4.2% to 0.8% over the past quarter — but actual fraud losses had increased.

**Task:** I was asked to investigate whether this was a model quality issue and what could have been detected earlier.

**Action:** I investigated two types of drift. Data drift: I compared the distribution of key model input features (transaction amount, device type, location) between the training data period and the current period. Transaction amounts had shifted significantly — the training data reflected pre-cost-of-living-crisis patterns, and actual transaction sizes had changed materially. The model had been trained on a world that no longer existed. Concept drift: I examined whether the relationship between inputs and fraud labels had changed — fraudsters had adopted new patterns (smaller, more frequent transactions) that the model had not been trained to recognise. I also reviewed what monitoring had been in place: the answer was none. The model's accuracy had never been measured in production against labelled outcomes. I recommended two mitigations: scheduled monthly retraining using recent labelled data, and a production monitoring job that compared the model's output distribution (fraud flag rate) against the historical baseline and alerted when it deviated by more than 20%.

**Result:** The model was retrained and fraud detection improved. The monitoring job was implemented and caught a further distribution shift six months later — before losses escalated. I documented drift as a standard risk in our AI feature launch checklist.

## Key Terms
| Term | Definition |
|------|------------|
| **Model drift** | The gradual degradation of model performance over time as real-world data diverges from the training data distribution |
| **Data drift** | A change in the statistical distribution of model input features in production compared to the training period |
| **Concept drift** | A change in the relationship between inputs and the target outcome — the world has changed in a way that makes the old model's learned patterns no longer valid |
| **Training data** | The labelled dataset used to train a machine learning model — its distribution defines the conditions under which the model is reliable |
| **Production monitoring** | Ongoing observation of a deployed model's behaviour — output distributions, performance metrics, and error rates — to detect drift and degradation |
| **Retraining** | The process of training a model again on more recent data to restore accuracy degraded by drift |
| **Output distribution** | The statistical pattern of a model's predictions over time — a shift in this distribution can signal drift even before labelled outcomes are available to measure accuracy directly |
