## The Intent
The interviewer is checking whether you can approach the testing challenge specific to generative AI — where there is no single correct answer to compare against. LLMs are increasingly embedded in products, and a QA engineer who can design a credible evaluation approach for them is genuinely differentiated.

## The STAR Script
**Situation:** My team shipped a customer support chatbot powered by an LLM. The QA suite at launch consisted of two tests: "does it return a response?" and "is the response under 500 characters?" Within a week of launch, users reported the bot confidently providing incorrect refund policy information.

**Task:** I was asked to design a more robust testing approach for the chatbot that would have caught the accuracy issue before launch.

**Action:** I designed a four-dimension evaluation framework. Accuracy: I created a golden test set of 50 questions with known correct answers drawn from our support documentation, and used an LLM-as-judge approach — prompting a separate LLM to score each response for factual accuracy against the source document on a 1–3 scale. Any response scoring 1 (incorrect) was flagged for human review. Relevance: I tested that responses addressed the question asked rather than drifting to related topics — again using an LLM judge prompted to score on-topic-ness. Safety and tone: I ran an adversarial prompt set — jailbreak attempts, requests to insult competitors, prompts asking for personal customer data — and verified that the model declined appropriately in all cases. Format compliance: I wrote deterministic assertions for output constraints the team had specified (no markdown bullet points in chat responses, no responses over 300 words) — these were fully automatable with regex and length checks.

**Result:** The golden test set approach caught an accuracy issue in the next model update — the LLM was answering questions about the old return policy after a policy change. The adversarial set caught a prompt injection vulnerability. Both were fixed before the update was released. I documented the framework and it was adopted for all subsequent LLM features.

## Key Terms
| Term | Definition |
|------|------------|
| **LLM (Large Language Model)** | A machine learning model trained on large text corpora capable of generating, summarising, classifying, and transforming natural language |
| **Non-deterministic output** | Output that varies between runs for the same input — characteristic of LLMs, requiring statistical and qualitative evaluation rather than exact-match assertions |
| **Golden test set** | A curated set of inputs with known expected outputs or quality criteria, used to evaluate model behaviour consistently across versions |
| **LLM-as-judge** | A technique where a second LLM is prompted to evaluate the quality of a first LLM's output against defined criteria — scalable automated quality scoring |
| **Adversarial prompting** | Deliberately crafted inputs designed to elicit unsafe, incorrect, or policy-violating responses from an LLM — used to test robustness and safety |
| **Prompt injection** | An attack where malicious content in user input attempts to override or hijack the LLM's system instructions |
| **Format compliance** | Verification that LLM output conforms to defined structural constraints — length, formatting, absence of prohibited content types |
