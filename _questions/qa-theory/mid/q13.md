question_id: 13
title: "What is the difference between static and dynamic testing?"
category: qa-theory
level: mid
theory: "Static testing examines software artefacts — such as requirements, design documents, and code — without executing the software. Dynamic testing involves executing the software and observing its behaviour. Both are essential and complementary; static testing prevents defects early while dynamic testing detects them in running systems."
---

## The Intent

The interviewer is assessing whether you actively participate in quality activities beyond test execution. Many mid-level testers are unaware of or undervalue static testing. Demonstrating fluency in both shows you understand quality as an end-to-end discipline, not just a testing phase.

---

## The STAR Script

**Situation:** Our team was experiencing a high rate of defects being found in system testing — many of which traced back to ambiguous or contradictory requirements written weeks earlier.

**Task:** I proposed introducing structured static testing activities earlier in the sprint cycle to catch these issues at the source.

**Action:** I introduced two static practices: **requirements reviews** at the start of each sprint (walking through user stories with the team to identify missing acceptance criteria, contradictions, and untestable statements) and **code reviews with a QA lens** (checking for error handling, edge case logic, and security basics before code was merged). These were lightweight — 30-minute sessions, not formal inspections. I tracked the defects found via static vs. dynamic means over the next month.

**Result:** 40% of defects in the following month were caught statically — before a single line of test code was run. The cost of fixing requirements-level issues in a review meeting was a fraction of the cost of fixing them post-development.

## Key Terms

| Term | Category | Description |
|------|----------|-------------|
| **Static Testing** | Static | Examining artefacts without executing code. |
| **Dynamic Testing** | Dynamic | Testing by running the software and observing behaviour. |
| **Informal Review** | Static | A lightweight peer check with no defined process. |
| **Walkthrough** | Static | Author-led review where team members ask questions. |
| **Inspection** | Static | Formal, structured review with defined roles and exit criteria. |
| **Static Analysis** | Static | Automated tool-based examination of code for defects (e.g., linting, SAST). |
| **SAST** | Static | Static Application Security Testing — automated code scanning for vulnerabilities. |
