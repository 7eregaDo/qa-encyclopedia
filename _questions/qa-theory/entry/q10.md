theory: "Functional testing verifies that a system behaves according to its specified functional requirements — i.e., what the system does. Non-functional testing evaluates the quality characteristics of how the system performs — e.g., how fast, how secure, how reliable, and how usable it is."

## The Intent

The interviewer wants to see that you understand testing encompasses more than just "does the feature work?" Many entry-level testers focus exclusively on functional correctness and overlook performance, security, and usability — which are often where the most damaging production issues originate.

---

## The STAR Script

**Situation:** I was testing a search feature for an e-commerce platform. The functional tests all passed — search returned the correct results for all test inputs.

**Task:** Before signing off, I needed to consider whether functional correctness alone was sufficient to declare the feature ready for production.

**Action:** I raised the question of non-functional coverage. We had not checked: how the search performed under load (performance), whether the search input was protected against SQL injection (security), how long the results took to appear (usability/performance), or how the feature behaved if the search service went down (reliability). I created a short non-functional checklist and ran basic checks for each area — a performance test with JMeter, a simple injection test, and a fallback scenario test.

**Result:** The injection test revealed a vulnerability in the search parameter handling. This was patched before release. The exercise led the team to adopt a non-functional acceptance checklist as a standard part of our definition of done.

## Key Terms

| Type | Category | Description |
|------|----------|-------------|
| **Functional Testing** | Functional | Verifies the system does what it is specified to do. |
| **Regression Testing** | Functional | Confirms existing functionality is unbroken after a change. |
| **UAT** | Functional | End-users verify the system meets their real-world needs. |
| **Performance Testing** | Non-Functional | Evaluates speed, scalability, and stability under load. |
| **Security Testing** | Non-Functional | Identifies vulnerabilities and validates access controls. |
| **Usability Testing** | Non-Functional | Assesses how easy and intuitive the system is to use. |
| **Reliability Testing** | Non-Functional | Tests the system's ability to function consistently over time. |
| **Compatibility Testing** | Non-Functional | Verifies the system works across different browsers, devices, and OS versions. |
