category: qa-theory

theory: "An error is a human mistake made during development. A defect (or bug) is the flaw in the code or artefact that results from that error. A failure is the observable incorrect behaviour that occurs when the defect is executed in the running system."

## The Intent

The interviewer is checking precision in your professional vocabulary. QA engineers communicate with developers, managers, and stakeholders — using these terms correctly (and consistently) prevents miscommunication and signals technical maturity beyond just "I find bugs."

---

## The STAR Script

**Situation:** A developer once pushed back on a ticket I raised, saying "that's not a bug, it's working as coded." The feature was calculating discounts incorrectly.

**Task:** I needed to clearly articulate the chain from human mistake to observable failure to justify the ticket without it becoming a debate.

**Action:** I walked through the terminology: the developer had misread the requirements document (the **error**), which led to incorrect logic being written into the discount function (the **defect**), which caused customers to be charged the wrong amount at checkout (the **failure**). Framing it this way separated the human mistake from the code problem and the user impact.

**Result:** The developer understood the distinction, accepted the defect, and the fix was prioritised. The structured vocabulary turned a potential conflict into a productive conversation.

## Key Terms

| Term | Definition |
|------|------------|
| **Error** | A human mistake — e.g., a developer misunderstanding a requirement. |
| **Defect / Bug** | A flaw in the code, design, or documentation resulting from an error. |
| **Failure** | The visible incorrect behaviour observed when a defect is triggered at runtime. |
| **Severity** | The technical impact of the defect on the system (e.g., Critical, Major, Minor). |
| **Priority** | How urgently the defect needs to be fixed from a business perspective. |
| **Defect Lifecycle** | The stages a defect moves through: New → Assigned → Fixed → Retested → Closed. |
