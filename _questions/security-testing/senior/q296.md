## The Intent
The interviewer wants to see whether your security thinking extends beyond the application code to the cloud environment it runs in. Many application security controls are rendered ineffective by misconfigured infrastructure — an open S3 bucket, an overly permissive IAM role, or disabled CloudTrail logging. Senior QEs who understand cloud security provide significantly broader coverage.

## The STAR Script
**Situation:** A production security incident at a company I joined had been caused by an EC2 instance with an overly permissive IAM role — the application had been compromised via SSRF, and the attacker used the instance's IAM role to access S3 buckets containing sensitive customer data. I was asked to lead an infrastructure security assessment to prevent similar incidents.

**Task:** Design and execute a cloud infrastructure security assessment covering IAM, storage, networking, secrets management, and logging.

**Action:** I conducted the assessment across five domains using a combination of AWS Config, Prowler, and manual review. IAM: I ran Prowler's IAM checks and reviewed all IAM roles attached to application services. Three roles had `*:*` wildcard policies — administrative permissions on application workload roles. I used IAM Access Analyzer to identify unused permissions and produced a least-privilege replacement policy for each role. I also verified that no IAM access keys existed for service accounts (instance profiles were used instead — correct). Storage: I checked all S3 buckets for public access settings, bucket policies allowing cross-account access, and server-side encryption. One bucket had a policy allowing unauthenticated read access — a legacy misconfiguration. All buckets now enforced AES-256 or KMS encryption. Networking: I reviewed security group rules. Two security groups had `0.0.0.0/0` ingress rules on ports other than 80/443 — one allowed SSH from the internet. I checked that the VPC had no unnecessary public subnets for application workloads. Secrets management: I confirmed all application secrets were in AWS Secrets Manager with automatic rotation enabled. I searched EC2 user data and Lambda environment variables for hardcoded secrets using a custom AWS CLI script. Logging: I verified CloudTrail was enabled in all regions with log file validation, that VPC Flow Logs were enabled, and that S3 access logging was active for sensitive buckets.

**Result:** Seven findings including the public S3 bucket (critical) and the overly permissive IAM roles (high). All were remediated within two weeks. The assessment was repeated quarterly thereafter using Prowler in the CI pipeline for IaC changes, and manually for a subset of controls.

## Seniority Shift
| Level | Expected Depth |
|-------|---------------|
| **Entry** | Aware that cloud configuration affects security; checks obvious settings like public S3 buckets when prompted |
| **Mid** | Checks IAM permissions, storage policies, and security group rules; uses IaC scanning (tfsec) in the pipeline |
| **Senior** | Designs the cloud security assessment across IAM, storage, networking, secrets, and logging; uses cloud-native tools (Prowler, AWS Config, IAM Access Analyzer); produces least-privilege IAM replacements; integrates cloud security scanning into CI |

## Key Terms
| Term | Definition |
|------|------------|
| **Prowler** | An open-source AWS security assessment tool that runs hundreds of checks against AWS configuration for compliance with CIS Benchmarks and security best practices |
| **IAM Access Analyzer** | An AWS service that identifies resources shared with external entities and unused permissions, used to identify overly permissive IAM policies |
| **Least Privilege IAM Policy** | An IAM policy granting only the specific permissions required for a workload, replacing wildcard policies that grant unnecessary access |
| **SSRF (Server-Side Request Forgery)** | A vulnerability where an attacker tricks the server into making HTTP requests to internal resources — in cloud environments, this can expose the instance metadata service and its IAM credentials |
| **Instance Metadata Service (IMDS)** | An AWS service accessible from EC2 instances that provides temporary IAM credentials — a common target in SSRF-based cloud attacks |
| **VPC Flow Logs** | AWS logs recording network traffic to and from resources within a VPC, used for security analysis and incident investigation |
| **CloudTrail** | An AWS service that records API calls made in an account, providing an audit trail for security investigations and compliance |
| **Security Group** | An AWS virtual firewall controlling inbound and outbound traffic to EC2 instances — overly permissive rules are a common cloud misconfiguration |
