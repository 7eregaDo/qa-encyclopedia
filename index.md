## 🎯 Quick Start by Role

**Just starting out in QA?**
Begin with [QA Theory → Entry]({{ site.baseurl }}/qa-theory/entry/) and [Manual Testing → Entry]({{ site.baseurl }}/manual-testing/entry/).

**Preparing for a mid-level interview?**
Focus on [Automation → Mid]({{ site.baseurl }}/automation/mid/), [API Testing → Mid]({{ site.baseurl }}/api-testing/mid/), and [Behavioural → Mid]({{ site.baseurl }}/behavioral/mid/).

**Going for a senior role?**
Cover [System Design → Senior]({{ site.baseurl }}/system-design/senior/), [CI/CD → Senior]({{ site.baseurl }}/ci-cd/senior/), and [Security Testing → Senior]({{ site.baseurl }}/security-testing/senior/).

**Targeting a Lead or Head of QA position?**
Start with [Behavioural → Lead]({{ site.baseurl }}/behavioral/lead/), [Automation → Lead]({{ site.baseurl }}/automation/lead/), and [Manual Testing → Lead]({{ site.baseurl }}/manual-testing/lead/).

## 📖 How to Use This Site

Each question page contains:
- **Intent** — what the interviewer is really testing
- **STAR Answer** — Situation, Task, Action, Result with real numbers
- **Seniority Shift** — how the expected answer changes by level
- **Badges** — `new` marks questions added beyond the original guide

Questions shared across multiple levels appear once and are linked from each relevant level page — no duplication.
